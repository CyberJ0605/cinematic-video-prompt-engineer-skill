# 电影感视频提示词工程师 Skill 项目交接文档

## 项目定位

`cinematic-video-prompt-engineer` 是一个“电影感视频提示词工程师”skill。

它的目标不是简单润色提示词，而是根据用户提供的剧情梗概、人物关系、场景设定和情绪方向，先诊断剧情，再把抽象剧情转译成适合 AI 视频模型生成的电影感视频提示词。

默认适用于 Kling、Seedance、Veo、Sora、Runway、即梦等通用 AI 视频模型。

## 默认工作方式

默认采用“打磨模式”，让用户能看到判断过程，方便继续调整。

输出结构：

```text
【剧情诊断】
情绪核心：
视觉核心：
结构判断：
时长判断：
取舍与补全：

【电影化改写策略】
...

【建议先生成的参考图】
人物参考图：
场景参考图：
关键道具参考图：

【最终视频提示词】
基础概括：
...
```

如果用户明确要求“直接给最终提示词”，可以省略诊断、策略和参考图。

## 适用范围

适合：

- 8-15 秒 AI 视频片段
- 情绪戏、亲密张力、悬疑、家庭冲突、回忆蒙太奇
- 人物头部特写和微表情表现
- 产品 / 人物质感短片
- 群像和大场面的 15 秒压缩
- 长剧情拆分成多条 15 秒提示词
- 用户满意第一条后，继续生成下一条视频提示词

不适合直接用于：

- 完整长片剧本
- 小说润色、标题、口播稿
- 一次性生成超过 15 秒的复杂剧情
- 缺少人物、场景、情绪目标的空泛需求
- 高复杂度武打动作设计，目前仍需参考样本继续打磨
- 依赖大量屏幕文字、字幕、UI 内容才能理解的画面
- 露骨性内容、未成年人性化、非自愿性内容、过度血腥内容

## 关键规则

### 时长

- 单条视频提示词最长不超过 15 秒。
- 不默认 15 秒，根据剧情判断合适时长。
- 如果 6 秒、8 秒、12 秒足够，就按实际时长输出。
- 如果剧情超过 15 秒承载能力，要说明本条提示词呈现哪一段，并建议拆分。
- 关键台词或动作不要压在最后一秒，结尾要留 1-2 秒给反应、呼吸、声音尾音或画面余韵。

### 长度

2000 字以内是最终视频提示词的上限，不是目标。

长度目标：

- 500-800 字：简单单人、单动作、单情绪场景
- 800-1300 字：默认范围，大多数 8-15 秒视频使用这一档
- 1300-2000 字：复杂场景，如多人对话、大场面、蒙太奇、长剧情拆分、空间动作

2000 字限制只针对【最终视频提示词】，不包括原始剧情、剧情诊断、电影化改写策略、参考图提示词和讨论内容。

### 模型字段

默认不单独输出：

```text
视频模型：通用 AI 视频模型
```

如果用户指定模型，再自然适配。

时长和结构放进 `基础概括` 第一句，例如：

```text
基础概括：这是一段 15 秒连续动作剪辑...
```

### 台词

如果剧情转折来自关键台词、电话、医生通知、警察通知、表白、分手、画外音或内心 OS，必须写出具体台词。

不要用模糊描述代替，例如不要只写：

```text
医生低声说完噩耗。
```

应写成：

```text
医生摘下口罩，低声说：“我们尽力了，但她没能撑过来。”
```

### 追问

只有缺少根基信息时才追问：

- 主角是谁
- 场景在哪里
- 情绪或转变是什么

缺少镜头、焦段、光影、声音、微表情等技术细节时，不追问，自动补全。

## 结构选择表

skill 已建立结构选择逻辑，先判断结构，再写镜头。

主要结构：

- 单场景一镜到底：一个空间，一个连续情绪变化
- 单场景连续剪辑：同一地点多个动作/反应节点
- 跳剪压缩：准备、决定、仪式、过程压缩
- 蒙太奇：记忆、梦境、象征、意识流
- 连续动作剪辑：追逐、逃跑、穿越空间
- 多人对话交叉剪辑：饭桌、谈判、争吵、审问
- 长特写微表情：靠脸部情绪递进撑住镜头
- 产品/人物质感片：车、手表、创始人、艺术家、场所
- 大场面压缩：灾难、宴会、战场、发布会、庭审
- 长剧情拆分：超过 15 秒承载能力时拆成多条
- 主观镜头：恐惧、眩晕、记忆触发、进入未知空间
- 匹配剪辑：过去/现在、童年/成年、旧身份/新身份呼应
- 剧情续写：从上一条尾帧继续

混合结构示例：

```text
主结构：多人对话交叉剪辑；辅助：长特写微表情
主结构：蒙太奇；辅助：匹配剪辑
主结构：剧情续写；辅助：尾帧参考
```

## 微表情与情绪规则

已建立“情绪到微表情映射表”，把抽象情绪转成可见动作。

覆盖情绪：

- 悲伤
- 震惊
- 强忍哭泣
- 愤怒克制
- 悔恨
- 愧疚
- 羞耻
- 羞怯
- 爱意克制
- 嫉妒
- 失望
- 释然
- 决绝
- 麻木
- 恐惧
- 复仇
- 阴冷
- 喜悦克制
- 尴尬

原则：

- 不直接写“她很悲伤”，要写眼神、嘴唇、下颌、喉结、呼吸、手指、肩膀。
- 长时间人物头部特写时使用微表情时间轴。
- 情绪变化要自然过渡，不能突然跳变。
- 复杂面部情绪戏可使用“超近面部长镜头情绪弧系统”。

## 超近面部长镜头情绪弧系统

该系统适用于全程靠脸部微表情演绎的镜头，不仅限于哭戏。

适用情绪：

- 悲伤
- 羞怯
- 爱意
- 愧疚
- 恐惧
- 嫉妒
- 黑化
- 决绝
- 释然
- 冷笑
- 喜悦克制

核心规则：

- 固定机位或极慢 Push in
- ECU/CU 超近面部特写
- 前半段尽量无肢体动作
- 全程无台词时，所有表演落在眼神、瞳孔、下眼睑、嘴角、双唇、泪水、呼吸
- 镜头可以代表另一个人，例如“镜头即她的心上人”
- 情绪必须形成波形，而不是平铺直叙

已经沉淀的两个模板：

```text
复杂悲伤弧：
震惊聚焦 -> 冷笑自嘲 -> 脆弱体面 -> 微笑瓦解 -> 大颗落泪 -> 低头叹息 -> 强撑释然 -> 抽泣反扑
```

```text
羞怯爱意弧：
突然看见 -> 呼吸一屏 -> 认出心上人 -> 眼神柔弯 -> 羞意上脸 -> 垂眼偷看 -> 压不住笑 -> 眼神弹开 -> 鼓起勇气重新直视
```

## 声音设计库

声音不是背景装饰，而是剧情和情绪结构的一部分。

已沉淀场景声音：

- 医院走廊：荧光灯、电流声、心电监护仪、护士脚步
- 雨夜车内：雨刷、车顶雨声、发动机怠速、转向灯
- 深夜公寓：冰箱低频、电话电流底噪、邻居脚步、电梯远声
- 厨房亲密戏：咖啡机、陶瓷杯、筷子碰碗、水管声
- 旧房间回忆：音乐盒发条、抽屉、纸张、童年笑声
- 火车站重逢：行李箱轮子、广播、铁轨风、空旷脚步回声
- 电梯轻喜剧：手机外放、电梯提示音、尴尬静默
- 古风宫殿：烛火、更鼓、衣袖摩擦、发簪细响
- 灾难大场面：警报、广播、金属扭曲、餐具滑落
- 产品/车：车门厚重关闭、点火、皮革摩擦、轮胎压碎石
- 非鬼怪悬疑：门锁、另一个房间的手机震动、地板轻响、呼吸
- 武侠动作：雨打竹叶、衣袖破风、金属接触点、剑出鞘、延迟雷声

规则：

- 优先真实环境声，不默认配乐。
- 关键台词不能被音乐盖住。
- 震惊时可以让声场变闷。
- 结尾要留声音尾巴。
- 如果不需要音乐，写明“无配乐，只保留环境声”。

## 负面约束库

按场景风险选择，不要机械堆长列表。

主要类别：

- 通用核心：不要字幕水印、脸部畸变、多余手指或肢体、卡通感
- 情绪特写：不要夸张哭喊、突然表情变化、过度磨皮、塑料皮肤
- 多人对话：不要舞台剧式表演、嘴型乱动、视线方向混乱
- 亲密张力：不要露骨性暗示、突然拥抱亲吻、偶像剧式夸张
- 悬疑恐惧：不要鬼怪、血腥、突脸惊吓、尖叫、夸张恐怖音乐
- 动作追逐：不要动作穿模、方向混乱、人物瞬移、过度动态模糊
- 武侠打戏：不要飞天玄幻、武器变形、多人动作混乱
- 群像灾难：不要人物数量失控、主角丢失、场景结构变形
- 产品/车：不要广告式夸张炫技、塑料质感、结构变形
- 古风：不要现代妆容、现代饰品、廉价影楼古风、游戏 CG 感
- 回忆梦境：不要过度梦幻发光、记忆画面过于完整清晰
- 手机/屏幕文字：不要乱码界面、错误文字、过多屏幕文字
- 餐桌/手部：不要手指畸形、餐具穿模、筷子变形、杯盘数量跳变

## 参考图提示词模块

新增了“建议先生成的参考图”模块，帮助用户先生成角色、场景和关键道具，再结合视频提示词生成视频。

目的：

- 提高人物一致性
- 提高场景一致性
- 稳定服装、发型、时代感、光线、关键道具
- 给用户一个更可控的工作流

规则：

- 这是可选模块，用户可以先生成参考图，也可以直接生成视频。
- 不强行给所有类别，只按剧情需要给。
- 参考图提示词必须和最终视频提示词保持一致。

常见组合：

- 单人情绪特写：人物参考图
- 对话/亲密戏：人物参考图 + 场景参考图
- 产品/车/人物质感片：产品或人物参考图 + 环境参考图
- 古风/时代戏：人物服装参考图 + 场景参考图
- 道具驱动剧情：关键道具参考图 + 场景参考图

## 续写能力

skill 已支持剧情续写。

触发语：

- 继续
- 接着往下写
- 延续上一条
- 下一段
- 下一镜
- 用上一条尾帧继续
- 第一条满意，写第二条

续写输出结构：

```text
【接续判断】
上一段结尾状态：
下一段情绪推进：
连续性注意：

【建议先生成的参考图】
沿用上一段尾帧/已有参考图：
新增人物参考图：
新增场景参考图：
新增关键道具参考图：

【下一段最终视频提示词】
基础概括：
...
```

续写规则：

- 从上一段尾帧开始，不重新开场。
- 人物身份、年龄、服装、发型、妆容、伤痕、情绪残留保持一致。
- 场景布局、光线、天气、时间、色调、声音环境保持一致。
- 关键道具保持一致。
- 情绪只能推进，不能重置。
- 每条 15 秒续写只推进一个新核心事件。
- 如果没有新视觉元素，说明“沿用上一段尾帧和已有参考图”。
- 如果出现新人物、新场景、新关键道具，新增对应参考图提示词。

## 已测试案例

已用以下类型测试过：

1. 极端情绪控制：深夜医院走廊，父亲听到母亲抢救失败，给女儿系鞋带憋泪。
2. 强动作与空间调度：暴雨夜少年追赶末班公交。
3. 意识流 / 回忆 / 梦境：女人打开旧音乐盒，现实与童年生日交错。
4. 亲密张力非露骨：分手后两人在清晨厨房同时拿同一只杯子。
5. 商业 / 产品 / 人物质感：年轻创始人读完投资人拒信，启动车离开。
6. 抽象主题转视觉：女孩镜前剪掉长发，与旧身份告别。
7. 古风人物头部特写：女子得知深爱之人将被赐死，烛前忍泪。
8. 多人对话：除夕饭桌上父亲宣布卖老房子。
9. 悬疑恐惧：独居女孩发现家里多了一把陌生钥匙，卧室传来手机震动。
10. 轻喜剧：电梯里辞职录音被老板听见。
11. 长剧情拆分：深夜老火车站母子重逢，拆成两个 15 秒。
12. 武侠打戏：雨夜竹林白衣剑客被围攻。
13. 群像 / 大场面压缩：暴风雨夜客轮餐厅混乱，母亲寻找孩子。
14. 激烈感情戏：雨夜车内情侣争执。
15. 古代宫廷情感戏：男主误闯浴殿，女主霸气回应，并续写下一段。
16. 超近面部黑化情绪弧：宫廷女子从受伤到冷硬决意。
17. 同人物从笑到哭：强撑笑意到彻底破防。

测试中发现并修正的问题：

- 关键台词不能省略。
- 结尾不能太突兀，要留余韵。
- `视频模型：通用 AI 视频模型` 是无效占位，已移除。
- 2000 字是上限，不是目标。
- 打戏目前效果不足，需要更多动作描写参考。
- 长特写微表情需要按情绪波形写，不只适用于哭戏。
- 参考图提示词能提升人物、场景、道具一致性。

## 安装与路径状态

当前用户希望 skills 统一放在：

```text
D:\Codex\skills
```

并通过目录链接让 Codex 默认路径指向 D 盘：

```text
C:\Users\HUAWEI\.codex\skills -> D:\Codex\skills
```

曾出现过 `cinematic-video-prompt-engineer` 嵌套同名目录，导致 `/` 菜单里出现两个同名 skill。

正确结构应为：

```text
D:\Codex\skills\cinematic-video-prompt-engineer\
  SKILL.md
  references\
    style_patterns.md
  agents\
    openai.yaml
```

不要出现：

```text
D:\Codex\skills\cinematic-video-prompt-engineer\cinematic-video-prompt-engineer\
```

## 维护建议

后续新增参考提示词时，不建议原样堆进 `SKILL.md`。

推荐方式：

- 核心流程和硬规则写入 `SKILL.md`
- 风格模板、情绪库、声音库、结构表写入 `references/style_patterns.md`
- 新样本先归纳成规则，再加入参考库
- 不确定是否通用的写法，先作为“实验模板”保留，不立刻变成硬规则

## 发布检查清单

每次向 GitHub 发布新版本时，必须同步完成：

1. 将工作区最新版 `SKILL.md` 和 `references/style_patterns.md` 同步到发布仓库中的 skill 目录。
2. 如果核心规则有变化，同步更新 `PROMPT_VERSION.md`，确保非 Codex 用户使用的通用版本不过期。
3. 在 `README.md` 的“更新日志”顶部新增版本条目，写明版本号、日期和本次主要变化。
4. 检查 README 案例、安装说明、作者信息和适用范围是否仍然准确。
5. 查看 `git diff --stat` 和 `git status`，确认没有遗漏或误提交文件。
6. 提交后推送到 GitHub，并确认本地分支不再显示 `ahead`。
